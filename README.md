## About Me

I am a **Computer Science student at The University of Texas at Dallas** who enjoys building practical software and studying how algorithms behave in real implementations.

My public projects currently focus on:

- **Machine learning:** classification, clustering, model comparison, and experiment design
- **Deep learning:** MLP and CNN image-classification pipelines
- **Software systems:** concurrent client-server applications and structured protocols
- **Algorithms:** graph traversal, route optimization, parsing, and data structures

I care about readable code, reproducible experiments, useful documentation, and clearly explaining how a system works.

### [Flight Path Finder](https://github.com/vinhthaitong/Flight-Path-Finder-)

A command-line Java application that models flights as a directed weighted graph and ranks routes by cost or travel time.

**Highlights**

- Builds an adjacency-list graph from pipe-delimited flight data
- Uses depth-first search to enumerate acyclic paths
- Tracks cumulative cost and duration for each route
- Supports cost-based and time-based optimization
- Produces readable route summaries and optimal-path results

## Engineering Approach

```text
Understand the problem
        ↓
Choose an appropriate model or data structure
        ↓
Build a clear implementation
        ↓
Validate behavior and measure results
        ↓
Document setup, decisions, and limitations
        ↓
Improve using evidence and feedback
```

I try to make every repository understandable to someone seeing it for the first time. A good project should explain not only **what the code does**, but also **how to run it**, **how it was evaluated**, and **where its limitations are**.
